# Codebase Structure

**Analysis Date:** 2026-04-02

## Directory Layout

```
chensoul.github.io/
├── content/            # Markdown/MDX content files (posts, briefs, translation, wiki)
├── public/             # Static assets served directly (images, favicons, data)
├── src/                # Source code (components, layouts, pages, utils)
│   ├── assets/         # SVG icons and static assets imported into components
│   ├── components/     # Reusable Astro components
│   ├── layouts/        # Page layout templates
│   ├── pages/          # Route definitions and API endpoints
│   ├── styles/         # Global and module CSS
│   └── utils/          # Utility functions and helpers
├── scripts/            # Node.js scripts (e.g., image conversion)
├── dist/               # Build output (generated by Astro)
├── docs/               # Project documentation
├── .github/            # GitHub Actions workflows
├── .planning/          # GSD planning documents
├── astro.config.ts     # Astro build configuration
├── content.config.ts   # Content Collections schema definitions
├── package.json        # Dependencies and scripts
├── pnpm-lock.yaml      # pnpm lockfile
├── tsconfig.json       # TypeScript configuration
├── eslint.config.js    # ESLint configuration
├── .prettierrc.mjs     # Prettier configuration
├── tailwind.config.ts  # Tailwind CSS configuration (v4 inline)
└── postcss.config.mjs  # PostCSS configuration
```

## Directory Purposes

**`content/`:**
- Purpose: Store all authored content as Markdown/MDX files
- Contains: Four content collections with date-prefixed filenames
- Key files:
  - `content/posts/`: Long-form blog posts (technology, life)
  - `content/briefs/`: Weekly notes and updates
  - `content/translation/`: Translated articles
  - `content/wiki/`: Knowledge base articles

**`public/`:**
- Purpose: Static assets served at root level
- Contains:
  - `public/images/`: Article images organized by post slug
  - `public/favicons/`: Site and category icons
  - `public/dist/`: Third-party CSS/JS (KaTeX, Artalk)
  - `public/data/`: Build-time generated data (feeds.json)

**`src/components/`:**
- Purpose: Reusable UI components
- Contains:
  - `Card.astro`: List item cards for all content types
  - `Header.astro`: Site navigation header
  - `Footer.astro`: Site footer with links
  - `ArticleToc.astro`: Table of contents for articles
  - `Artalk.astro`: Comment system integration
  - `Photosuite.astro`: Image gallery component
  - `Tag.astro`: Tag pill component
  - `SocialIcon.astro`: Social media icon component
  - `BackToTop.astro`: Scroll-to-top button
  - `YearProgress.astro`: Year progress indicator

**`src/layouts/`:**
- Purpose: Page structure templates
- Contains:
  - `Layout.astro`: Base layout with HTML shell, meta tags, theme
  - `PostDetails.astro`: Article detail page layout
  - `ListLayout.astro`: List page layout for collections

**`src/pages/`:**
- Purpose: Route definitions and API endpoints
- Contains:
  - `index.astro`: Homepage
  - `[section]/[...slug]/index.astro`: Dynamic article pages
  - `[section]/[...slug].md.ts`: Markdown API route
  - `posts/index.astro`, `briefs/index.astro`, etc.: Collection index pages
  - `tags/[tag]/[...page].astro`: Tag archive pagination
  - `og/[...slug].png.ts`: OG image generation
  - `rss.xml.ts`, `sitemap.xml.ts`, `llms.txt.ts`: Feed endpoints
  - `about.astro`, `links.astro`, `running.astro`: Static pages
  - `search.astro`: Search page with Pagefind
  - `404.astro`: Not found page

**`src/utils/`:**
- Purpose: Business logic and helper functions
- Contains:
  - `postUtils.ts`: Post path resolution, tag extraction, description generation
  - `contentCollections.ts`: Unified content collection utilities
  - `articleTime.ts`: Date formatting with timezone support
  - `blogImages/`: Image CDN logic and Markdown image processing
  - `og-image.ts`: Social image generation with Satori
  - `legacyRedirects.ts`: 301 redirect mapping for migrated content
  - `quietAstroMermaid.ts`: Mermaid diagram integration

**`src/assets/`:**
- Purpose: Imported static assets (mainly SVG icons)
- Contains: `src/assets/icons/` with SVG icon components

**`src/styles/`:**
- Purpose: Global and module CSS
- Contains:
  - `global.css`: Tailwind imports and theme configuration
  - `base.css`: CSS variables and global reset
  - `taxonomy-pill.css`: Tag/category pill styles
  - `pages/running.css`: Running page specific styles

## Key File Locations

**Entry Points:**
- `src/pages/index.astro`: Homepage rendering
- `src/pages/[section]/[...slug]/index.astro`: Article detail pages
- `astro.config.ts`: Build configuration

**Configuration:**
- `src/config.ts`: Site-wide configuration (SITE object)
- `src/content.config.ts`: Content Collections schema definitions
- `tsconfig.json`: TypeScript path aliases (`@/*` → `./src/*`)

**Core Logic:**
- `src/utils/postUtils.ts`: Post utilities class
- `src/utils/blogImages/index.ts`: Image URL resolution logic

**Testing:**
- No test framework configured (static site)

## Naming Conventions

**Files:**
- Pages: `[name].astro` or `[name].ts` for API routes
- Components: PascalCase (e.g., `ArticleToc.astro`)
- Utils: camelCase (e.g., `postUtils.ts`)
- Content: `YYYY-MM-DD-title.md` with date prefix
- Styles: kebab-case (e.g., `taxonomy-pill.css`)

**Directories:**
- Lowercase, plural for collections (`components/`, `layouts/`, `utils/`)
- Singular for content types (`posts/`, `briefs/`, `translation/`, `wiki/`)

## Where to Add New Code

**New Feature (e.g., new page type):**
- Primary code: `src/pages/{feature}.astro`
- Component (if needed): `src/components/{Feature}.astro`
- Utility (if needed): `src/utils/{feature}.ts`

**New Content Type:**
- Add collection in `src/content.config.ts`
- Create content directory: `content/{name}/`
- Add index page: `src/pages/{name}/index.astro`
- Update `getAllBlogLike()` in `src/utils/contentCollections.ts`

**New Utility Function:**
- Create or extend file in `src/utils/`
- Export from module for use in pages/components

**New Component:**
- Implementation: `src/components/{Component}.astro`
- Use Astro component syntax with frontmatter
- Scope styles with `<style>` block

**New Style:**
- Global: `src/styles/base.css` or new file in `src/styles/`
- Component-scoped: `<style>` block inside `.astro` component
- Import in `src/styles/global.css` if needed globally

## Special Directories

**`dist/`:**
- Purpose: Build output directory
- Generated: Yes (by `astro build`)
- Committed: No (in `.gitignore`)

**`.astro/`:**
- Purpose: Astro runtime cache and generated types
- Generated: Yes (by Astro dev/build)
- Committed: No (in `.gitignore`)

**`node_modules/`:**
- Purpose: npm/pnpm dependencies
- Generated: Yes (by `pnpm install`)
- Committed: No (in `.gitignore`)

**`public/`:**
- Purpose: Static assets copied directly to build output
- Generated: Partially (some files committed, some build-time)
- Committed: Yes (except build-time generated files)

## Path Aliases

TypeScript path alias configured in `tsconfig.json`:

```json
{
  "@/*": ["./src/*"]
}
```

Usage in code:
```typescript
import { SITE } from "@/config";
import Layout from "@/layouts/Layout.astro";
```
