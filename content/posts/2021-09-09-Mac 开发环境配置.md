新买的 Mac 或者刚重装完系统，如何快速恢复到熟悉的开发环境？本文记录我从零开始配置 macOS 开发环境的完整流程，所有命令和配置都已实践验证。

## 环境信息

- **系统**：macOS 15.6.1 (Darwin 24.6.0)
- **芯片**：Apple M1 (arm64)
- **时间**：2026 年 4 月

## 第一步：系统初始化工具

### 1.1 安装 Xcode Command Line Tools

这是 macOS 开发的基础，提供 Git、Clang 等必要工具：

```bash
xcode-select --install
```

弹出对话框后点击「安装」即可。验证安装：

```bash
git --version
# git version 2.39.5 (Apple Git-154)
```

### 1.2 macOS 系统设置（可选）

`install.sh` 脚本包含以下系统优化设置，可根据需要选择执行：

**系统名称和时区：**

```bash
# 设置计算机名
sudo scutil --set ComputerName "chensoul-mac"
sudo scutil --set HostName "chensoul-mac"
sudo scutil --set LocalHostName "chensoul-mac"

# 设置时区
sudo systemsetup -settimezone "Asia/Shanghai"
```

**安全和键盘设置：**

```bash
# 取消 4 位数密码限制
sudo pwpolicy -clearaccountpolicies

# 允许安装任意来源的应用
sudo spctl --master-disable

# 加快键盘重复速度
defaults write NSGlobalDomain KeyRepeat -int 1
defaults write NSGlobalDomain InitialKeyRepeat -int 10
```

**滚动条和动画：**

```bash
# 显示滚动条
defaults write NSGlobalDomain AppleShowScrollBars -string "WhenScrolling"

# 禁用不必要的动画（提升响应速度）
defaults write com.apple.dock launchanim -bool false
defaults write NSGlobalDomain NSAutomaticWindowAnimationsEnabled -bool false
defaults write com.apple.dock expose-animation-duration -float 0.1

# 加快对话框显示速度
defaults write NSGlobalDomain NSWindowResizeTime -float 0.001
```

**Finder 设置：**

```bash
# 显示文件扩展名
defaults write NSGlobalDomain AppleShowAllExtensions -bool true

# 显示路径栏和状态栏
defaults write com.apple.finder ShowPathbar -bool true
defaults write com.apple.finder ShowStatusBar -bool true

# 默认搜索当前文件夹
defaults write com.apple.finder FXDefaultSearchScope -string "SCcf"

# 禁用创建 .DS_Store 文件（网络卷和 USB）
defaults write com.apple.desktopservices DSDontWriteNetworkStores -bool true
defaults write com.apple.desktopservices DSDontWriteUSBStores -bool true
```

**应用更改：**

```bash
# 重启 Dock 和 Finder 使设置生效
killall Dock
killall Finder
```

> **提示**：以上设置已集成到 `install.sh` 脚本中，运行脚本会自动应用。

### 1.3 安装 Homebrew

macOS 的包管理器，后续所有软件都通过它安装：

**国内用户推荐使用镜像源加速访问：**

```bash
# 使用清华大学镜像源安装 Homebrew
export HOMEBREW_INSTALL_FROM_ZIP=1
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

安装完成后，配置镜像源环境变量（添加到 `~/.zprofile` 或 `~/.config/fish/config.fish`）：

```bash
# Homebrew 镜像源（清华大学）
export HOMEBREW_API_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api"
export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"
export HOMEBREW_CDN_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"
export HOMEBREW_PIP_INDEX_URL="https://pypi.mirrors.ustc.edu.cn/simple"

# 应用配置
echo 'export HOMEBREW_API_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles/api"' >> ~/.zprofile
echo 'export HOMEBREW_BOTTLE_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"' >> ~/.zprofile
echo 'export HOMEBREW_CDN_DOMAIN="https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles"' >> ~/.zprofile
```

> **提示**：如果使用 USTC 镜像源，将 `mirrors.tuna.tsinghua.edu.cn` 替换为 `mirrors.ustc.edu.cn` 即可。

然后将 Homebrew 添加到 PATH：

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

验证：

```bash
brew --version
# Homebrew 4.x.x
```

## 第二步：配置 Dotfiles

我使用 [**chezmoi**](https://www.chezmoi.io/) 管理所有配置文件，它支持 Git 版本控制和 GPG 加密。

### 2.1 克隆 Dotfiles 仓库

```bash
mkdir -p ~/github/dotfiles
git clone https://github.com/chensoul/dotfiles.git ~/github/dotfiles
cd ~/github/dotfiles
```

### 2.2 安装 Homebrew 软件

仓库中的 `Brewfile` 包含了所有需要安装的软件：

```bash
brew bundle --file=Brewfile
```

这会一次性安装：

- **Shell 工具**：fish、starship、zoxide
- **开发语言**：node、python、maven、sdkman
- **命令行增强**：bat、eza、fd、ripgrep、delta
- **开发工具**：chezmoi、gnupg、just、jq
- **字体**：font-jetbrains-mono-nerd-font

### 2.3 应用配置文件

```bash
chezmoi init --apply ~/github/dotfiles
```

如果有 GPG 加密文件，会提示输入密码。

### 2.4 切换默认 Shell

```bash
# 将 fish 设为默认 shell
chsh -s /opt/homebrew/bin/fish
```

重启终端后，提示符会从 `user@MacBook ~ %` 变成带颜色图标的样式。

> **提示**：Nerd Font 已通过 Brewfile 自动安装，无需手动执行。Starship 的 Catppuccin Powerline 主题需要手动应用（见第三步）。

## 第三步：终端配置

### 3.1 安装 Ghostty 终端

[Ghostty](https://ghostty.org/) 是一款现代化的终端模拟器：

```bash
brew install --cask ghostty
```

打开 Ghostty，它会自动读取 `~/.config/ghostty/config` 配置：

```ini
# 窗口大小（Ghostty 1.3+）
window-width = 120
window-height = 40

# 字体（必须先安装 Nerd Font）
font-family = "JetBrainsMono Nerd Font"
font-size = 14

# macOS 特定
macos-option-as-alt = true

# 透明度
background-opacity = 0.95

# 滚动历史（100MB）
scrollback-limit = 100000000

# 关闭确认
confirm-close-surface = true
```

> **注意**：Ghostty 1.3+ 版本移除了 `theme` 和 `bell-features` 等选项，配置文件已相应更新。

### 3.2 安装 Nerd Font

Starship 的图标需要 Nerd Font 支持：

```bash
brew install --cask font-jetbrains-mono-nerd-font
```

重启 Ghostty 后生效。

### 3.3 Starship 提示符

Starship 已随 Brewfile 安装。我使用 **Catppuccin Powerline** 主题，采用权力线（Powerline）风格的彩色分隔块效果。

**应用主题：**

```bash
starship preset catppuccin-powerline -o ~/.config/starship.toml
```

**效果示例：**

```
 chensoul  ~/github/chensoul.github.io   main   24.14.0   10:30  in 1.234s
❯
```

**主题特点：**

| 颜色块 | 内容                          |
|-----|-----------------------------|
| 红色  | 操作系统 + 用户名                  |
| 橙色  | 当前目录                        |
| 黄色  | Git 分支 + 状态                 |
| 绿色  | 语言版本（Node.js、Java、Python 等） |
| 蓝色  | Conda 环境                    |
| 紫色  | 时间 + 命令执行时长                 |

**配色方案：**

配置文件使用 Catppuccin Mocha（深色）配色，支持 4 种变体：

- **Mocha** - 深色主题（默认）
- **Frappe** - 中等对比度
- **Latte** - 浅色主题
- **Macchiato** - 柔和深色

> **提示**：配置文件已包含完整的颜色定义，无需额外安装主题。必须安装 Nerd Font 才能正确显示图标。

## 第四步：开发语言环境

### 4.1 Node.js 和 pnpm

使用 fnm 管理 Node 版本：

```bash
# 安装最新版 Node.js
fnm install latest
fnm default latest

# 验证
node -v    # v24.14.0
pnpm -v   # 10.33.0
```

项目级别的版本通过 `.nvmrc` 自动切换。

### 4.2 Java 和 Maven

使用 SDKMAN 管理 Java 版本：

```bash
# 安装 Java 17
sdk install java 17.0.11-tem

# 安装 Java 8（旧项目需要）
sdk install java 8.0.482-amzn

# 设置默认版本
sdk default java 17.0.11-tem

# 安装 Maven
sdk install maven
```

验证：

```bash
java -version
# openjdk version "17.0.11"
mvn -version
# Apache Maven 3.9.12
```

### 4.3 Python 和 uv

```bash
# uv 是快速的 Python 包管理器
brew install uv python@3.14

# 创建虚拟环境
uv venv
source .venv/bin/activate

# 安装包
uv add requests pytest
```

## 第五步：开发工具配置

### 5.1 编辑器

安装 Cursor（VS Code 分支，AI 增强）：

```bash
brew install --cask cursor
```

安装 IntelliJ IDEA：

```bash
brew install --cask intellij-idea
```

### 5.2 Git 增强

安装 Delta（Git diff 增强）：

```bash
brew install git-delta
```

配置 `~/.gitconfig`：

```ini
[core]
    pager = delta

[interactive]
    diffFilter = delta --color-only

[delta]
    navigate = true
    side-by-side = true
    line-numbers = true

[merge]
    conflictstyle = diff3
```

### 5.3 命令行增强

这些工具的别名已配置在 `fish/config.fish`：

| 原生命令   | 替代工具  | 说明        |
|--------|-------|-----------|
| `cat`  | `bat` | 语法高亮      |
| `ls`   | `eza` | 图标、颜色、树状图 |
| `find` | `fd`  | 更快、更友好    |
| `grep` | `rg`  | 极速搜索      |

## 第七步：个性化配置

所有 Shell 别名和函数都已整合到 `fish/config.fish` 中。

### 7.1 实用函数

**批量拉取代码：**

```bash
pullcode          # 拉取 ~/github 和 ~/work 下所有仓库
pullcode .        # 只拉取当前目录下的一级子目录
```

**批量提交代码：**

```bash
pushcode "chore: sync"         # add → commit → push 一站式
pushcode --here "fix: typo"    # 只处理当前目录
```

**清理构建产物：**

```bash
cleanup          # 删除 node_modules, target, dist 等
cleanup --dry-run  # 预览将删除的内容
```

**杀死占用端口的进程：**

```bash
killport 3000    # 释放 3000 端口
```

**其他实用函数：**

| 函数       | 说明                     |
|----------|------------------------|
| `y`      | Yazi 文件管理器（cd 到退出时的目录） |
| `touchd` | 快速创建文件（自动创建父目录）        |
| `proj`   | 快速进入 `~/github/项目名`    |
| `fd`     | 查找并进入目录                |

## 附录：Chezmoi 管理文件清单

### 哪些文件应该用 Chezmoi 管理？

根据我的实践经验，以下文件适合用 chezmoi 管理：

#### 必须管理的配置文件

| 文件/目录                        | 说明                    | chezmoi 命名                             |
|------------------------------|-----------------------|----------------------------------------|
| `~/.myrc`                    | 个人自定义配置（可选）           | `private_dot_myrc`                     |
| `~/.zprofile`                | 登录 Shell 环境变量         | `private_dot_zprofile`                 |
| `~/.gitconfig`               | Git 全局配置              | `executable_dot_gitconfig`             |
| `~/.ssh/config`              | SSH 客户端配置             | `dot_ssh/config`                       |
| `~/.config/fish/config.fish` | Fish Shell 配置（含别名和函数） | `dot_config/fish/config.fish`          |
| `~/.config/starship.toml`    | Starship 提示符          | `dot_config/starship.toml`             |
| `~/.config/ghostty/config`   | Ghostty 终端            | `dot_config/ghostty/config`            |
| `~/.config/yazi/`            | Yazi 文件管理器            | `dot_config/yazi/`                     |
| `~/.vscode/settings.json`    | VS Code 设置            | `dot_config/Code/User/settings.json`   |
| `~/.cursor/settings.json`    | Cursor 设置             | `dot_config/Cursor/User/settings.json` |

#### 可选管理的配置文件

| 文件/目录                | 说明                        |
|----------------------|---------------------------|
| `~/.wakatime.cfg`    | WakaTime 配置（含 API 密钥，需加密） |
| `~/.m2/settings.xml` | Maven 镜像配置                |
| `~/.rclone.conf`     | Rclone 远程存储配置（需加密）        |
| `~/.npmrc`           | NPM 配置（含 token 需加密）       |
| `~/.zshrc`           | Zsh 配置（如果还在使用）            |
| `~/.tmux.conf`       | Tmux 配置                   |
| `~/.vimrc`           | Vim 配置                    |

#### 不应该管理的文件

| 文件/目录                         | 原因            |
|-------------------------------|---------------|
| `~/.DS_Store`                 | macOS 自动生成    |
| `~/.zsh_history`              | Shell 历史记录    |
| `~/.bash_history`             | Shell 历史记录    |
| `~/.local/state/`             | 应用运行时状态       |
| `~/.cache/`                   | 缓存目录          |
| `~/.Trash/`                   | 回收站           |
| `~/Library/`                  | 系统库目录         |
| `~/.ssh/id_ed25519`           | SSH 私钥（应单独备份） |
| `~/.gnupg/private-keys-v1.d/` | GPG 私钥（应单独备份） |

### 加密敏感文件

对于包含密钥、密码的文件，使用 GPG 加密：

```bash
# 加密文件
chezmoi add --encrypt ~/.wakatime.cfg

# 解密查看
chezmoi cat ~/.wakatime.cfg

# 编辑加密文件
chezmoi edit ~/.wakatime.cfg
```

加密后的文件以 `.asc` 后缀存储在 dotfiles 仓库中，只有拥有 GPG 私钥才能解密。

### 我的 Dotfiles 目录结构

```
dotfiles/
├── Brewfile                      # Homebrew 软件清单（含镜像源配置）
├── install.sh                    # 一键安装脚本（含 macOS 系统设置）
├── README.md                     # 说明文档
├── .chezmoiignore                # 忽略文件列表
├── private_dot_zprofile          # 登录 Shell 环境变量（最小配置）
├── dot_config/
│   ├── fish/config.fish          # Fish Shell 配置（含别名和函数）
│   ├── starship.toml             # Starship（Catppuccin Powerline 主题）
│   ├── ghostty/config            # Ghostty 终端配置
│   └── rclone/                   # Rclone 配置
├── dot_ssh/
│   └── config                    # SSH 配置
├── encrypted_dot_wakatime.cfg.asc # WakaTime 配置（加密）
└── executable_dot_gitconfig      # Git 配置
```

运行：

```bash
cd ~/github/dotfiles
bash install.sh
```

> **注意**：脚本中的计算机名、GitHub 用户名等需要根据实际情况修改。

## 总结

按照以上步骤，大约 30 分钟可以完成从零到可用的开发环境配置。

### 核心思路

1. **配置即代码**：所有配置文件版本化，可重现
2. **工具现代化**：fish、eza、bat、fd 提升体验
3. **自动化优先**：能用脚本不手动
4. **安全加密**：敏感信息 GPG 加密
