你是不是也配过环境变量、搭过开发环境，后来又离不开 Copilot、Cursor？

写代码的人换了一茬又一茬，手里的家伙却最有故事。从记事本和 Vim，到重型 IDE，再到 Git、容器和今天的 AI 补全——这篇把**工具链**按年代串成一条线，方便你对照自己是在哪一段上车的。

不是教科书，也不是厂商软文；主观回忆和二手说法难免有偏差，当博客读就好。产品名、版本以各官网为准。

**阅读提示：** 适合手机碎片化浏览，按小标题跳读即可；下面列表偏「检索向」，收藏后需要时翻一翻。

## 1990s - 编辑器时代

**经典编辑器的黄金年代**

- **Vim** — Bram Moolenaar，基于 Stevie

- **Emacs** - GNU Emacs 成为自由软件运动标杆

- **UltraEdit, EditPlus** - Windows 平台早期文本编辑器

**典型工作流：** 手写代码，手动编译；**版本控制尚未在多数小团队中普及**；此前已有 SCCS、RCS、CVS 等，但未像后来 Git 那样成为默认习惯

## 2000s - IDE 时代

**集成开发环境崛起**

- **Eclipse** — IBM 开源，Java 开发标配

- **Visual Studio** - .NET 开发首选

- **IntelliJ IDEA** — JetBrains，智能代码补全

- **NetBeans** - Sun Microsystems 开源 IDE

**版本控制主流化：**

- **Subversion**，常称 **SVN** — 集中化版本控制普及

- **Git** 开始在小圈子流行

- **GitHub** 成立

**典型工作流：** IDE 开发，SVN 版本控制，手动部署

## 2010-2014 - 轻量级编辑器崛起

- **Sublime Text** — 轻量、快、插件生态；ST2 前后带动普及

- **Atom** — GitHub, Electron

- **Visual Studio Code** — 微软 Build 公开预览起

- **Git** 在新建与开源项目中逐渐成为默认选项之一

- **GitHub** 成为最大代码托管平台

**典型工作流：** 编辑器 + Git + 手动测试

## 2015-2018 - DevOps 时代

- **Docker** — 容器化普及

- **Kubernetes** — 容器编排广泛采用

- **Jenkins** — CI/CD 标配

- **VS Code** 正式版

**前端革命：**

- npm/yarn 包管理器

- Webpack/Babel 构建工具

- React/Vue/Angular 框架

**典型工作流：** 轻量编辑器 + Git + Docker + CI/CD

## 2019-2020 - 云原生时代

- **GitHub Actions** — CI/CD 集成到 GitHub

- **GitLab CI/CD** 成熟

- **VS Code Remote** — 远程开发

- **GitHub Codespaces** — 云端开发环境

- **TypeScript** 在大型前端项目中广泛采用

**典型工作流：** 云端开发，自动化测试，自动部署

## 2021 - AI 编程元年

- **GitHub Copilot** 技术预览

- **VS Code** 在 Stack Overflow 等开发者调查中常居编辑器前列

- **Docker** 进入企业主流

- **GitLab CI/CD** 工具链标准化

**小记：** VS Code、Git、Docker、Chrome DevTools 仍是日常底座。

## 2022 - 商业化元年

- **ChatGPT** 发布，引爆 AI 热潮

- **GitHub Copilot** 个人版正式发布（2022 年），订阅制

- **AI 代码审查 / 补全类工具** — CodeGPT、Tabnine、CodeWhisperer 等

- **TypeScript** 在大型前端项目中广泛采用，但整体用量尚未超过 JavaScript

**小记：** Copilot、ChatGPT、VS Code；工作流多是手写一段再让 ChatGPT 补、改、粘。

## 2023 - AI 编程 1.0 时代

- **ChatGPT** 据报道用户规模迅速攀升，程序员中广泛使用

- **Stack Overflow** 访问量承压，据报道与 AI 辅助编程等多重因素相关，不宜单归因

- **Copilot X** 公布

- **生成式 UI**：**v0**、**Bolt**、**Lovable**、**Replit** 等同赛道见下节「生成式应用」

国内厂商和模型很多，下面按类型扫一眼，**列举无排名**，名称以各厂商为准：

**国内 AI 编码**（2023-2024 年主流产品，排名不分先后）：

- **云与互联网**：通义灵码、文心快码、豆包、腾讯云代码助手、华为云 CodeArts / 智能编程、讯飞 / iFlyCode、智谱 CodeGeeX、蚂蚁 CodeFuse、商汤小浣熊、京东言犀、网易伏羲……

- **模型与 API**：DeepSeek、百川、MiniMax、月之暗面 Kimi、零一万物、面壁、阶跃星辰……

- **平台与生态**：魔搭 ModelScope、硅基流动、各云模型广场 / API 网关……

- **AI 编辑器与智能体**：Trae、各云编程智能体、多文件改写……

- **企业侧 AI 编码试点** 增多，规模因公司与行业而异

**小记：** 问 → 粘代码 → 改 → 报错再问，循环变常见。

## 2024 - AI 原生 IDE 崛起

- **AI 原生 / 强 AI 编辑器**：Cursor、Windsurf、Zed、Trae、JetBrains AI Assistant、Continue; Amazon Q Developer、Google 系插件 / IDX / Antigravity、GitHub Copilot 深度集成等

- **生成式应用 / 全栈原型**：v0、Bolt、Lovable、Replit、Framer、[Builder.io](http://Builder.io)、Plasmic、[21st.dev](http://21st.dev)…… 以各官方为准

- **国内** — 上列各厂产品与模型路线并存，与 Cursor 等海外工具同属常见选型

- **多文件编辑成熟** — AI 开始理解项目结构

- **Agent 概念普及** — 从被动回答到主动执行

**小记：** Cursor / Windsurf / Zed、国内各云助手与 DeepSeek 等模型 API、v0 / Bolt / Lovable / Replit；在仓库里和 AI 一起改文件比纯问答多一步。

## 2025 - Agent 能力爆发

- **终端 / 仓库级 Agent**: Claude Code、GitHub Copilot / Copilot CLI、Aider、Amazon Q CLI、Gemini CLI、OpenAI Codex 系、Warp…… 命名与形态变化快，以官方为准

- **生成式应用 / 编辑器** — v0、Bolt、Lovable、Replit、Cursor、Windsurf 等持续迭代

**小记：** Claude Code、Copilot CLI、Aider、Gemini CLI 等；终端里下一句指令跑一轮改造，已能见到实例。

## 2026 - 全流程自动化时代进行中

下面几条是**当前正在进行**的趋势，变化很快，看看就好，别当成已经普遍落地的定论。

- **多 Agent 协作系统** 成熟

- **移动端 AI 编程** 成熟 — Rork、VibeCode、CodeFlying 等

- **预测性编程** — AI 预判下一步操作

- **全流程自动化** — 从需求到运维

**小记：** 多 Agent、Cursor、Claude Code、Copilot、v0 / Bolt 等名字还会换；当趋势与新闻看可以，别当成已普遍落地的定论。

## 技术范式演进

从「手写」到「人机协作」，大致可以这么理解——**最后几条更像方向**，别当已经发生。

- **1990s–2000s**：**手工编程**，编写代码 → 手动编译 → 手动调试 → 手动部署

- **2010s**：**工具链化**，编辑器 → 版本控制 → 自动测试 → 持续集成

- **2021-2022**：**辅助编程**，人类主导 → AI 建议 → 人类决策

- **2023**：**对话编程**，人类提问 → AI 回答 → 人类执行

- **2024**：**协同编程**，人类 + AI → 实时协作 → 共同决策

- **2025**：**Agent 主导**，人类设定目标 → AI 规划 → AI 执行 → 人类审核

- **2026**：**AI 军团**，人类指挥 → 多 Agent 协作 → 全自动完成

能做到哪一步，还是看团队和业务。

## 写在最后

工具换得快，焦虑也容易跟着涨。说实话，不必被「不会用 AI 就淘汰」吓到，但**新工具确实能少写一点重复劳动**——前提是你知道自己要写什么、怎么验收。

装哪几个软件没有标准答案。能把需求说清楚、能一眼看出代码对不对的人，总归少踩坑。
