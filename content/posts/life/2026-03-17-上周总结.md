**4、每日简报**

阅读了 [我让龙虾替我工作了 38 天，它做了什么？](https://atbug.com/openclaw-38-days-what-did-it-do/)
之后，研究了一下作者开发的 [openclaw skills](https://github.com/addozhang/openclaw-forge)。

这篇文章挺有意思的。作者让 OpenClaw 工作了 38 天，然后复盘了三个 agent 的产出：

Nova（数字管家）

• 维护开源项目、回复 issue
• 从零到发布完成 Nexus MCP Server
• 开发了一批 skill：daily-briefing、google-tasks、tech-doc-translator、research-prep、quick-capture
• 批量提交 44 篇博客到微软 MVP Portal

Forge（编程搭档）

• 重写 Android 断食追踪 App（Kotlin + Jetpack Compose）
• 开发 Feed Pulse（Telegram RSS 订阅机器人）
• 参与技术决策和可行性分析

Assistant（打杂）

• 协助 GitHub Copilot Dev Days 广州筹备
• 帮作者太太整理 HR 数据

其中有一个 daily-briefing 的
agent，每天早 7 点自动推送天气、待办、Hacker News Top 20 到 Telegram。

这和我在 github action 中使用 python 开发的[每日简报](https://github.com/chensoul/chensoul/blob/main/daily_review.py)
很像，于是我修改了我的脚本，发送的每日简报包括日期、天气(wttr.in)、今日待办(GitHub Issues)
、今日指数/诗词/名言/Trending、WakaTime、跑步距离、昨日收藏(Linkding)、Hacker News等等。

今日待办我没有使用 Google Tasks，开始打算使用 memos（使用 todo） 作为代办任务，后来放弃了，想起了可以将 github issues
作为代办任务。也许后续还会使用其他提供 API 接口的代办任务工具。

**为什么不使用 openclaw 来实现每日简报？** 对于每日简报这种比较确定的任务，使用 github action 完全就可以实现，而且也不需要 AI
功能，所以可以不用使用 openclaw 来实现，免得消耗 token。
