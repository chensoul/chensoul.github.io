本文记录 **重装 macOS 前后** 的 checklist。日常 shell 与软件清单以私有仓库 **`dotfiles`** 及其 README 为准（文中命令与仓库若有出入，以仓库为准）；路径、邮箱、远程地址请按你的环境改写。

## 重装 macOS 之前

抹盘或「抹掉所有内容与设置」前，先确认**数据与授权**已在其他介质或云端有副本，避免重装后缺文件、缺密钥。

### 1. 备份文件与密钥（iCloud 云盘）

统一备份到 **iCloud 云盘**（访达侧栏「iCloud 云盘」与下列路径一致）。重装后同一 Apple ID 登录即可同步。

```bash
ICLOUD="$HOME/Library/Mobile Documents/com~apple~CloudDocs"
BACKUP="${ICLOUD}/backup/mac-reinstall"
mkdir -p "$BACKUP"
```

**合规**：公司代码、密钥是否允许进入 iCloud 须遵守你方政策；极敏感内容请用加密移动硬盘或密码管理器。

- **代码目录**（示例为 `~/github`、`~/work`）

  打包前可用 `cleanup` 清理常见构建目录与 `*.log`、`.DS_Store`（定义在 `~/.aliases`；`cleanup -n` 可先预览不删除）。若无该函数，可手动删大目录后再打包。

  ```bash
  cd ~
  cleanup
  zip -r "${BACKUP}/github.zip" ~/github
  zip -r "${BACKUP}/work.zip" ~/work
  ```

- **本地 Maven 仓库**（体积大，也可重装后重新下载依赖）

  ```bash
  zip -r "${BACKUP}/m2.zip" ~/.m2
  ```

- **SSH 密钥**

  ```bash
  cp ~/.ssh/id_ed25519 "${BACKUP}/"
  cp ~/.ssh/id_ed25519.pub "${BACKUP}/"
  ```

  私钥**不要**提交到 Git；是否放入 iCloud 请自行权衡风险。

- **可选：`~/.myrc`**（环境变量、TOKEN，若使用）

  ```bash
  [[ -f ~/.myrc ]] && cp ~/.myrc "${BACKUP}/"
  ```

- **GPG**（Git 签名、Maven Central 发布等）

  自动取密钥环中**第一把**私钥并导出，上传公钥到密钥服务器：

  ```bash
  cd "$BACKUP"
  KEY_ID=$(gpg --list-secret-keys --with-colons 2>/dev/null | awk -F: '/^sec:/{print $5; exit}') && \
  test -n "$KEY_ID" && \
  gpg --armor --export-secret-keys "$KEY_ID" > "gpg-private-${KEY_ID}.asc" && \
  gpg --armor --export "$KEY_ID" > "gpg-public-${KEY_ID}.asc" && \
  gpg --keyserver keys.openpgp.org --send-keys "$KEY_ID"
  ```

  文件保存在 **`${BACKUP}/`**，文件名含 `KEY_ID`。导出私钥时若提示口令，属正常。

### 2. 应用配置导出

对无法仅靠重装恢复的软件提前导出，例如 **cc-switch**、**TablePlus**、**SwitchHosts**、**Clash Verge**（订阅与规则）等，将导出文件放入 **`${BACKUP}/`** 或子目录。**勿**将含密钥的导出提交到公开仓库。

### 3. 软件清单（Homebrew）

在「软件已装齐」的本机导出清单，便于对照：

```bash
ICLOUD="$HOME/Library/Mobile Documents/com~apple~CloudDocs"
BACKUP="${ICLOUD}/backup/mac-reinstall"
mkdir -p "$BACKUP"
brew bundle dump --force --file="${BACKUP}/Brewfile"
```

若日常以 `dotfiles` 里的 `Brewfile` 为准，也可不 dump，直接依赖仓库版本。

### 4. 其他

- 密码管理器（如 1Password）确认 **主密码 / Secret Key** 在新环境可用。  
- **转手或出售**：退出 iCloud、App Store、iMessage 后再抹盘；若曾在 iCloud 存密钥或代码包，在其他设备上删除或移走敏感内容。

## 重装 macOS 之后

以下顺序兼顾 **先能连上 Git**，再 **装工具链**。

### 1. 系统与账号

登录 Apple ID，打开 **iCloud 云盘** 同步；设置时区、输入法；若公司要求再开 FileVault 等。

### 2. Command Line Tools

```bash
xcode-select --install
```

未完成前不要执行 `install.sh`（脚本内也会检查）。

### 3. 从 iCloud 恢复备份

同步完成后，备份目录一般为：

`~/Library/Mobile Documents/com~apple~CloudDocs/backup/mac-reinstall/`

```bash
ICLOUD="$HOME/Library/Mobile Documents/com~apple~CloudDocs"
BACKUP="${ICLOUD}/backup/mac-reinstall"

mkdir -p ~/.ssh
chmod 700 ~/.ssh
cp "${BACKUP}/id_ed25519" ~/.ssh/ 2>/dev/null || true
cp "${BACKUP}/id_ed25519.pub" ~/.ssh/ 2>/dev/null || true
chmod 600 ~/.ssh/id_ed25519 2>/dev/null || true

gpg --import "${BACKUP}"/gpg-private-*.asc 2>/dev/null || true
gpg --import "${BACKUP}"/gpg-public-*.asc 2>/dev/null || true

[[ -f "${BACKUP}/myrc" ]] && cp "${BACKUP}/myrc" ~/.myrc

# 按需解压（路径按你备份时的 zip 名称）
unzip "${BACKUP}/m2.zip" -d ~
unzip "${BACKUP}/github.zip" -d ~
unzip "${BACKUP}/work.zip" -d ~
```

若无 SSH 备份，则之后在新环境执行 `ssh-keygen`，并把**新公钥**加到 GitHub/GitLab。GPG 若未备份，只能新建密钥并在托管平台更新公钥。

### 4. 克隆 dotfiles 并安装环境

私有仓库建议用 **SSH**（需已恢复 `~/.ssh` 或使用 HTTPS + Token）：

```bash
git clone git@github.com:chensoul/dotfiles.git
cd dotfiles
chmod +x install.sh link-dotfiles.sh
```

任选其一：

- **一键**：`bash install.sh`（系统 `defaults`、Homebrew、`brew bundle`、SDKMAN、按需生成 SSH、`link-dotfiles.sh` 等，见仓库 README）。若 **`~/.ssh/id_ed25519` 已存在**，脚本不会覆盖。  
- **仅软件与点文件**：`brew bundle install --file=Brewfile`，再 `bash link-dotfiles.sh`。

**务必**使用 `bash install.sh`，不要使用 `sh install.sh`。

若尚未有 `~/.myrc`，可从仓库复制模板后编辑：

```bash
cp .myrc.example ~/.myrc
```

### 5. SDKMAN / Java

按 `sdk list` 安装常用 JDK、Maven；`install.sh` 中的版本号若过期，在脚本或终端中自行更新。

### 6. 网络与代理

恢复 **Clash Verge** 等客户端与订阅；若 `.gitconfig` 中为 GitHub 配置了代理，确认本机代理端口与之一致。

### 7. 应用与插件

用重装前的导出恢复 **cc-switch**、**TablePlus**、**SwitchHosts**、**Clash Verge** 等；无备份则只能手工重配。

**IntelliJ IDEA** 插件示例：`.ignore`、Atom Material Icons、Laconic POM for Maven、Maven Helper、Rainbow Brackets。  

**Chrome** 插件示例：JsonVue、1Password。

### 验收

```bash
java -version
node -v   # 若使用 fnm，按 dotfiles 中说明加载
git st
gpg -K    # 确认密钥已导入（若使用签名）
```

再 `git clone` 一个**私有仓库**，确认 SSH 与（若使用）代理正常。

## 附

- **dotfiles**：`Brewfile`、`.zprofile`、`.aliases`、`.gitconfig`、`gitignore_global` 等由仓库维护；`~/.myrc` 仅存本机密钥，勿提交。  
- **GPG 与 Maven Central**：发布构件需配置 `maven-gpg-plugin`，公钥需可在密钥服务器查询；CI 签名请用 GitHub Actions Secrets，勿将私钥写入仓库。
