## 顶级缓存策略

**从系统读取数据**：
- Cache Aside（旁路缓存）
- Read Through（读穿透）

**向系统写入数据**：
- Write Around（写绕过）
- Write Back（写回）
- Write Through（写穿透）

## 译者总结

这篇文章非常简短，但列出了缓存系统的核心策略：

**读策略**：
- **Cache Aside**：应用先查缓存，未命中再查数据库并回填缓存（最常用）
- **Read Through**：缓存层自己负责从数据库加载，对应用透明

**写策略**：
- **Write Around**：直接写数据库，绕过缓存（适合写多读少）
- **Write Through**：同时写缓存和数据库，保证一致性（适合读多写少）
- **Write Back**：先写缓存，异步刷到数据库（性能最高，但有数据丢失风险）

**组合使用**：
- Cache Aside + Write Through：常见组合，读走旁路，写穿透保证一致
- Cache Aside + Write Back：高性能场景，接受短暂不一致

实际选择取决于：
- 读写比例
- 一致性要求
- 性能需求
- 可接受的数据丢失风险

理解这些策略的权衡对设计高效缓存系统至关重要。
