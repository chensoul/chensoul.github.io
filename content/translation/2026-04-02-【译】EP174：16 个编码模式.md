## 如何学习数据库

数据库为从网站和应用到企业系统的一切提供动力。以下是帮助你掌握数据库的学习路线图：

### 1. 数据库基础
包括"什么是数据库"、RDBMS、SQL vs NoSQL、ACID vs BASE、OLTP vs OLAP、事务和隔离级别等主题。

### 2. 数据模型和类型
包括关系数据库、非关系数据库和数据类型（整数、字符串、布尔、日期、JSON 等）等主题。

### 3. 查询和语言
包括 SQL 基础（SELECT、INSERT 等）、高级 SQL（视图、索引、CTE 等）和 NoSQL 查询（聚合和键值查找）等主题。

### 4. 索引和优化
包括索引（B-Tree、哈希和位图）、查询执行计划、去规范化 vs. 规范化、分片、连接池和查询批量等主题。

### 5. 安全、备份和扩展
包括用户角色、权限、加密、SQL 注入、高可用性（复制和故障转移）、水平 vs. 垂直扩展等主题。

### 6. 工具和生态系统
包括流行 SQL 数据库、NoSQL 数据库、GUI 工具、ORM、云数据库服务（RDS、DynamoDB、Google Cloud SQL 等）等主题。

## 当你在浏览器中输入 URL 时会发生什么

让我们逐步查看过程。

### 步骤 1：用户输入 URL 并按 Enter
首先需要做的是将 URL 转换为 IP 地址。映射通常存储在缓存中，所以浏览器在多层缓存中查找 IP 地址：浏览器缓存、OS 缓存、本地缓存和 ISP 缓存。如果浏览器在缓存中找不到映射，它将要求 DNS（域名系统）解析器解析它。

### 步骤 2：递归 DNS 查找
如果在任何缓存中都找不到 IP 地址，浏览器去 DNS 服务器进行递归 DNS 查找直到找到 IP 地址。

### 步骤 3：发送 HTTP 请求
现在我们有了服务器的 IP 地址，浏览器向服务器发送 HTTP 请求。为安全访问服务器资源，我们应该始终使用 HTTPS。它首先通过 TCP 三次握手与服务器建立 TCP 连接。然后它发送公钥到客户端。客户端使用公钥加密会话密钥并发送到服务器。服务器使用私钥解密会话密钥。客户端和服务器现在可以使用会话密钥交换加密数据。

### 步骤 4：服务器处理和响应
服务器处理请求并发送回响应。对于成功响应，状态码是 200。响应中有 3 部分：HTML、CSS 和 Javascript。浏览器解析 HTML 并生成 DOM 树。它还解析 CSS 并生成 CSSOM 树。然后它组合 DOM 树和 CSSOM 树以渲染树。浏览器渲染内容并显示给用户。

## HTTPS 如何工作

HTTPS 使用加密确保你的浏览器（客户端）和网站服务器之间的安全通信。

### 1. TCP 握手
在任何安全通信发生之前，客户端和服务器首先使用 TCP 握手过程建立基本连接。此时，数据未加密。

### 2. 证书检查
此阶段旨在验证服务器身份。客户端用"hello"消息开始 TLS 握手，提供支持的加密算法。服务器回复，选择算法并发送其数字证书，其中包含公钥。客户端验证证书以确保它来自受信任的证书颁发机构（CA）。

### 3. 密钥交换
一旦客户端验证证书，它开始密钥交换过程。客户端使用服务器的公钥（来自证书）加密会话密钥。加密的会话密钥发送到服务器，服务器可以使用其私钥解密它。更改密码规范消息表示从此时起，所有消息将使用商定的会话密钥和加密加密。此步骤使用非对称加密安全交换会话密钥。

### 4. 数据传输
现在，双方切换到使用共享会话密钥的对称加密（更快）。消息用相同密钥加密和解密。所有交换的数据现在安全且私密。

## Netflix 的扩展演进

多年来，Netflix 经历了几个增长阶段。每个阶段，他们不得不演进扩展方法。

### 阶段 1（早期流媒体架构）
这是典型的 3 层，由客户端、API 和数据库组成。API 应用名为 NCCP（Netflix Content Control Protocol）。它是包含所有功能的单体应用。

### 阶段 2（引入微服务）
随着添加更多功能，维护工程速度变得关键。为支持此，单体应用被分解为微服务。功能从 NCCP 应用提取并作为独立应用开发，但编排逻辑保留在 NCCP 应用中。

### 阶段 3（Zuul 网关）
接下来，Netflix 拆分应用。NCCP 保留用于播放体验，而其他 API 开始处理其他功能。引入 Zuul API 网关以减少客户端和服务之间的耦合。

## 译者总结

这篇文章涵盖了编码模式、数据库学习、URL 流程、HTTPS 和 Netflix 扩展的核心概念：

| 双指针 | 数组/链表操作 |

| 滑动窗口 | 子数组问题 |

| 栈 | LIFO 操作 |
| 堆 | 优先队列 |
| 前缀和 | 范围求和 |
| 树 | 层次数据 |
| 字典树 | 字符串前缀 |
| 图 | 网络关系 |
| 回溯 | 组合问题 |
| 动态规划 | 最优子结构 |
| 贪心算法 | 局部最优 |
| 区间 | 区间重叠 |

**数据库学习 6 大领域**：
| 领域 | 内容 |
|------|------|
| 基础 | RDBMS、SQL vs NoSQL、ACID |
| 数据模型 | 关系、非关系、数据类型 |
| 查询 | SQL、NoSQL 查询 |
| 索引优化 | B-Tree、执行计划、分片 |
| 安全备份 | 权限、加密、复制 |
| 工具生态 | SQL/NoSQL 数据库、ORM、云服务 |

**URL 输入浏览器 4 步骤**：
1. DNS 解析（多层缓存）
2. 递归 DNS 查找
3. HTTP/HTTPS 请求（TCP 握手）
4. 服务器响应（HTML/CSS/JS 解析渲染）

**HTTPS 4 步骤握手**：
1. TCP 握手（未加密）
2. 证书检查（验证服务器）
3. 密钥交换（非对称加密）
4. 数据传输（对称加密）

**Netflix 扩展 3 阶段**：
| 阶段 | 架构 | 特点 |
|------|------|------|
| 1 | 3 层单体 | 客户端、API、数据库 |
| 2 | 微服务 | 功能提取、编排保留 |
| 3 | Zuul 网关 | 减少耦合 |

**关键洞察**：
- 编码模式帮助解决面试问题
- 数据库学习需要系统性方法
- URL 流程涉及 DNS 和 HTTP
- HTTPS 结合非对称和对称加密
- Netflix 从单体演进到微服务

理解这些概念对编码面试和系统设计至关重要。
