GenAI 指能够创建新内容的系统，如文本、图像、代码或音乐，通过学习现有数据的模式。以下是 GenAI 技术栈的关键构建块：

## GenAI 技术栈组件

### 1. 云托管和推理
像 AWS、GCP、Azure 和 Nvidia 这样的提供商提供运行和扩展 AI 工作负载的基础设施。

### 2. 基础模型
核心 LLM（如 GPT、Claude、Mistral、Llama、Gemini、Deepseek）在海量数据上训练，形成所有 GenAI 应用的基础。

### 3. 框架
像 LangChain、PyTorch 和 Hugging Face 这样的工具帮助构建、部署和将模型集成到应用中。

### 4. 数据库和编排
向量数据库（如 Pinecone、Weaviate）、编排工具（如 LangChain、LlamaIndex）管理记忆、检索和逻辑流。

### 5. 微调
像 Weights & Biases、OctoML 和 Hugging Face 这样的平台支持为特定任务或领域训练模型。

### 6. 嵌入和标注
像 Cohere、Scale AI、Nomic 和 JinaAI 这样的服务帮助生成和标注向量表示，为搜索和 RAG 系统供电。

### 7. 合成数据
像 Gretel、Tonic AI 和 Mostly AI 这样的工具创建人工数据集以增强训练。

### 8. 模型监督
监控模型性能、偏见和行为。工具如 Fiddler、Helicone 和 WhyLabs帮助。

### 9. 模型安全
帮助确保 GenAI 系统的道德、安全和安全部署。解决方案如 LLM Guard、Arthur AI 和 Garak 帮助这个。

## 数据库索引类型

数据库索引是派生结构，将列值映射到表中行的物理位置。让我们看看一些关键索引类型：

### 主索引
当表上定义主键时自动创建此索引。这种索引可以是密集的以及稀疏的，尽管在大多数场景中首选稀疏。

- **密集索引**：包含表中每行的一个条目
- **稀疏索引**：只包含表中一些行的条目

### 聚簇索引
聚簇索引确定表中行的物理顺序。表上只能存在一个聚簇索引，因为数据只能以一种顺序存储。它适合范围查询、有序扫描和 I/O 效率。

### 辅助索引
非聚簇索引是单独的结构，保存一个或多个列的副本以及指向表中实际行的指针。它不影响数据如何物理存储，可以使用主索引定位记录。

## AI 代理学习路线图

- **AI 代理**：能够自主行动的系统，对其环境做出反应，使用工具（API、互联网、代码等），并可以在人类指导下工作
- **构建 AI 代理**：必须知道工具如 Python、Jupyter、PyTorch 和 GitHub Copilot。这些支持编码、实验和与 AI 库和 API 集成
- **GenAI 基础模型**：熟悉大型模型如 GPT、Gemini、LLaMa、DeepSeek 和 Claude 至关重要。这些模型提供基础智能，代理可用于推理、生成和理解
- **AI 代理开发栈**：工具如 Langchain、AutoGen、Crew AI 和框架如 Semantic Kernel 和 Hugging Face 为代理工作流供电。这些组件管理任务、记忆和外部工具集成在代理管道中
- **API 设计**：理解 API 设计方法如 REST、GraphQL、gRPC 和 SOAP 对构建可互操作代理至关重要。关键概念包括 HTTP 方法、状态码、版本控制、cookie、头部和缓存
- **AI 代理类型**：了解几种类型的 AI 代理，如简单反射、基于模型的反射、基于目标的、基于效用的和学习代理。每个在复杂性上不同
- **AI 代理系统架构**：AI 代理可以作为单代理、在多代理系统中或在人机协作中操作。架构取决于用例

## 12 个 MCP 服务器

MCP（模型上下文协议）是一个开放标准，简化 AI 模型（特别是 LLM）如何与外部数据源、工具和服务交互。MCP 服务器充当这些 AI 模型和外部工具之间的桥梁。以下是顶级 MCP 服务器：

1. **文件系统 MCP 服务器**：允许 LLM 直接访问本地文件系统来读取、写入和创建目录
2. **GitHub MCP 服务器**：连接 Claude 到 GitHub 仓库，允许文件更新、代码搜索
3. **Slack MCP 服务器**：Slack API 的 MCP 服务器，使 Claude 能够与 Slack 工作空间交互
4. **Google Maps MCP 服务器**：Google Maps API 的 MCP 服务器
5. **Docker MCP 服务器**：与 Docker 集成以管理容器、镜像、卷和网络
6. **Brave MCP 服务器**：使用 Brave 的搜索 API 进行 Web 和本地搜索
7. **PostgreSQL MCP 服务器**：一个 MCP 服务器，使 LLM 能够检查数据库模式并执行只读查询
8. **Google Drive MCP 服务器**：与 Google Drive 集成的 MCP 服务器，允许读取和搜索文件
9. **Redis MCP 服务器**：提供访问 Redis 数据库的 MCP 服务器
10. **Notion MCP 服务器**：这个项目为 Notion API 实现 MCP 服务器
11. **Stripe MCP 服务器**：与 Stripe API 交互的 MCP 服务器
12. **Perplexity MCP 服务器**：连接到 Perplexity 的 Sonar API 进行实时搜索的 MCP 服务器
