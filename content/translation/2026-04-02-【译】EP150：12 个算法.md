## 系统设计面试的 12 个算法

以下是系统设计面试中你应该知道的 12 个算法：

1. **布隆过滤器（Bloom Filter）**
   在执行磁盘操作之前检查请求的项是否在缓存中。

2. **Geohash**
   用于构建基于位置的服务。

3. **HyperLogLog**
   计数用户执行的唯一查询。

4. **一致性哈希（Consistent Hashing）**
   用于集群节点之间的高效数据分布。

5. **Merkle 树（Merkle Tree）**
   用于检测多个节点间数据副本的不一致性。

6. **Raft 算法（Raft Algorithm）**
   用于在日志复制上达成共识。

7. **Lossy Count**
   网络流量分析、Web 分析和识别重击者。

8. **四叉树（QuadTree）**
   用于构建基于位置的服务。

9. **操作转换（Operational Transformation）**
   用于支持协作编辑系统。

10. **漏桶（Leaky Bucket）**
    用于速率限制。

11. **Rsync**
    在两个不同系统之间同步文件和目录。

12. **光线投射（Ray Casting）**
    用于地理空间分析、视频游戏中的碰撞检测和计算机图形学。

## Kubernetes 如何工作

Kubernetes（K8S）是一个开源容器编排平台，最初由 Google 开发，现在由云原生计算基金会（CNCF）维护。

### 开发者如何与 Kubernetes 交互

- 开发者创建描述应用的清单文件。
- Kubernetes 获取这些清单文件，验证它们，并在其 worker 节点集群上部署应用。
- Kubernetes 管理应用的整个生命周期。

### Kubernetes 的两大组件

#### 1. 控制平面（Control Plane）
它像 Kubernetes 的大脑，由以下部分组成：

- **API 服务器（API Server）**：接收所有来自用户或 CLI 的传入请求。
- **调度器（Scheduler）**：基于资源可用性和约束决定在哪里运行新 pods。
- **控制器管理器（Controller Manager）**：确保集群的期望状态与实际状态匹配。
- **Etcd**：Kubernetes 保存所有数据的键值存储，包括配置、集群状态和应用期望状态。

#### 2. Worker 节点（Worker Nodes）
它们是集群中的 worker。每个节点有以下子组件：

- **Kubelet**：每个节点上的主要 worker。它与 API 服务器通信，获取指令，并确保容器按预期运行。
- **Kube-proxy**：处理每个节点上的网络。

## PostgreSQL 架构

使用 C 语言构建，PostgreSQL 是从小型 Web 应用到企业系统的最流行数据库选择。它作为多进程系统运行，遵循 ACID 原则。

### PostgreSQL 架构的关键点

- **并发客户端连接**：PostgreSQL 独立支持并发客户端连接。每个到 PostgreSQL 的客户端连接创建一个专用的服务器进程。

- **Postmaster 进程**：是管理所有其他 PostgreSQL 进程的主管。它控制整个数据库实例。

- **后台 worker**：在需要时运行并行进程以处理专门任务。

- **PostgreSQL 共享内存**：是包含多个缓冲区的中央内存区域，如共享、WAL、Clog 和临时缓冲区。所有组件通过这个共享内存通信。

- **PostgreSQL 还有几个辅助进程**：
  - BG Writer：管理后台写入
  - WAL Writer：处理预写日志
  - Auto Vacuum：维护数据库清洁
  - Checkpointer：确保数据一致性
  - Stats Collector：收集统计
  - System Logger：管理日志
  - Archiver：处理归档
  - Replication launcher：管理复制

- **PostgreSQL 有不同类型的物理文件用于不同需求**：
  - 数据文件：存储实际数据库数据
  - WAL 文件：预写日志存储
  - 归档文件：备份和恢复数据
  - 日志文件：系统和错误日志
