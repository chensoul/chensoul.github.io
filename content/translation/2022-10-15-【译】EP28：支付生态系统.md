## 支付生态系统

金融科技初创公司如何在众多支付公司中找到新机会？PayPal、Stripe 和 Square 具体做什么？

下图展示了支付生态系统的各个环节：

### 步骤 0-1：发卡流程
- 持卡人在发卡行开设账户并获得借记卡/信用卡
- 商家向 ISO（独立销售组织）或 MSP（成员服务提供商）注册以进行店内销售
- ISO/MSP 与支付处理器合作开设商家账户

### 步骤 2-5：收单流程
- 支付网关接受购买交易并收集支付信息
- 然后发送到支付处理器，处理器使用客户信息收集付款
- 收单处理器将交易发送到卡网络
- 它还拥有并在结算期间运营商家账户（这不是实时发生的）

### 步骤 6-8：发卡流程
- 发卡处理器代表发卡行与卡网络通信
- 它验证并运营客户账户

## Web 1.0 / Web 2.0 / Web 3.0

下图从鸟瞰图角度展示了 Web 1.0/Web 2.0/Web 3.0：

### Web 1.0 - 只读（1991-2004）
互联网就像一个静态页面目录。我们可以通过从一个超链接跳到另一个来浏览内容。它不提供与内容的任何交互。

### Web 2.0 - 读写（2004 至今）
互联网已发展到拥有搜索引擎、社交媒体应用和推荐算法支持的应用。

由于这些应用数字化人类行为并在用户与应用交互时持久化用户数据，大公司利用用户数据进行广告投放，这成为 Web 2.0 的主要商业模式之一。

这就是为什么人们说应用比你的朋友、家人甚至你自己更了解你。

### Web 3.0 - 读写拥有
由于区块链和去中心化应用的发展，这个想法最近被广泛讨论。创作者的内容存储在 IPFS（星际文件系统）上并由用户拥有。如果应用想访问数据，它们需要从用户获得授权并为此付费。

在 Web 3.0 中，所有权变更可能导致一些重大创新。
