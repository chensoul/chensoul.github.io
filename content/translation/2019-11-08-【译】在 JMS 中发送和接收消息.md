本文将介绍 JMS 架构，并演示如何在 JMS 中发送和接收消息。内容涵盖管理对象（Administrated Objects）、JMS 1.1 API 和 JMS 2.0 API。你将看到 JMS 2.0 API 相比 JMS 1.1 API 有多么简洁。在开始之前，请确保已完成 JMS 环境的设置。如果你是初学者，建议先理解 [JMS 基础](https://jstobigdata.com/jms/jms-introduction-java-message-service/)。

- 在 JMS 中工作时，`ConnectionFactory` 和 `Destination` 对象（Queue 和 Topic）需要在 JMS 提供者中配置。因此它们被称为**管理（Administered）**对象。
- 通常通过 JNDI 查找获取 `ConnectionFactory`，然后从中获取物理的 `Connection` 对象。
- 获取 Connection 后，需要打开一个 `Session` 来发送/接收消息。

## JMS API 版本

主要有两套接口用于发送和接收消息：

- **JMS 1.1** - 也称为 **经典 API（Classic API）**，这是遗留 API。
- **JMS 2.0** - 称为 **简化 API（Simplified API）**，所需接口更少。这是新项目应该使用的版本。

### 公共接口

- **Queue** - 用于点对点消息传递的管理对象。单个生产者和单个接收者。
- **Topic** - 用于发布/订阅消息的管理对象。单个生产者，多个接收者。
- **Destination** - Queue 和 Topic 的公共超类型。
- **Message、ByteMessage、MapMessage、ObjectMessage、StreamMessage** - 表示发送到 JMS 提供者或从 JMS 提供者接收的消息类型。

## 使用 JMS 发送和接收消息

下面分别使用 JMS 1.1（经典 API）和 JMS 2.0（简化 API）演示简单的文本消息发送和接收示例。推荐使用 JMS 2.0 API，这也是本教程后续的重点。但了解旧 API 对于处理遗留应用也很有帮助。

### 使用 JMS 1.1 开发客户端

以下示例演示经典 API 的用法。它创建一个简单的文本消息并发送到 Queue，然后由消息消费者从同一 Queue 读取该消息。

```java
package lab00.classic.helloworld;

import javax.jms.*;
import javax.naming.InitialContext;
import javax.naming.NamingException;

public class TestHelloWorldQueue {
    public static void main(String[] args) throws NamingException {
        InitialContext initialContext = null;
        try {
            initialContext = new InitialContext();
            // Step-1: Create ConnectionFactory
            ConnectionFactory connectionFactory =
                (ConnectionFactory) initialContext.lookup("jms/__defaultConnectionFactory");
            // Step-2: Create connection
            Connection connection = connectionFactory.createConnection();
            // Step-3: Create Session (可从 Connection 创建多个 Session)
            Session session = connection.createSession();
            // Step-4: Get the Queue
            Queue queue = (Queue) initialContext.lookup("jms/PTPQueue");
            // Step-5a: Create the message Producer
            MessageProducer messageProducer = session.createProducer(queue);
            // Step-6a: Create the Text Message
            TextMessage textMessage = session.createTextMessage("Test message - Hello");
            // Step-7a: Send the message
            messageProducer.send(textMessage);
            // Step-5b: Create the message Consumer
            MessageConsumer messageConsumer = session.createConsumer(queue);
            // Step-6b: Start the connection
            connection.start(); // to start delivery
            // Step-7b: Receive the message
            TextMessage message = (TextMessage) messageConsumer.receive();
            System.out.println(message.getText());
        } catch (NamingException | JMSException ex) {
            ex.printStackTrace();
        } finally {
            if (initialContext != null) {
                initialContext.close();
            }
        }
    }
}
```

上述代码展示了使用 JMS 1.1 发送和接收消息的步骤：

1. 使用 JNDI 查找创建 `ConnectionFactory`。JNDI 信息在 `jndi.properties` 文件中配置。每个应用/服务只需一个 ConnectionFactory。
2. 从 ConnectionFactory 获取 `Connection` 对象。
3. 从 Connection 创建 `Session`。
4. 使用 Session 创建 `MessageProducer` 和 `MessageConsumer` 对象。
5. 调用 Connection 的 start 方法开始传递消息。
6. 最重要的是，记得在 finally 块或 [try-with-resources](https://docs.oracle.com/javase/tutorial/essential/exceptions/tryResourceClose.html) 块中**关闭** Session、Connection、MessageProducer 和 MessageConsumer 对象。（示例中为简洁起见省略了这一步。）

### 使用 JMS 2.0 开发客户端

下面使用 JMS 2.0 完成同样的任务：向 Queue 发送问候消息并由消费者读取。注意：以下示例仅用于展示 JMS 2.0 的基本用法。实际项目中应参考本教程后续部分更合适的代码示例。

```java
package lab00.simple.helloworld;

import javax.jms.*;
import javax.naming.InitialContext;
import javax.naming.NamingException;

public class TestHelloWorld {
    public static void main(String[] args) {
        ConnectionFactory connectionFactory = null;
        Queue queue = null;
        try {
            InitialContext initialContext = new InitialContext();
            // Step-1: Create ConnectionFactory
            connectionFactory =
                (ConnectionFactory) initialContext.lookup("jms/__defaultConnectionFactory");
            // Step-2: Get the Queue
            queue = (Queue) initialContext.lookup("jms/PTPQueue");
        } catch (NamingException e) {
            e.printStackTrace();
        }
        // Step-3: Create JMSContext
        try (JMSContext jmsContext = connectionFactory.createContext()) {
            // Step-4a: Send message
            TextMessage textMessage = jmsContext.createTextMessage("Message using JMS 2.0");
            JMSProducer jmsProducer = jmsContext.createProducer().send(queue, textMessage);
            // Step-4b: Receive message
            TextMessage message = (TextMessage) jmsContext.createConsumer(queue).receive();
            System.out.println(message.getText());
        } catch (JMSException e) {
            e.printStackTrace();
        }
    }
}
```

JMS 2.0 的步骤更加直观：

1. 获取 `ConnectionFactory`。与经典 API 不同，无需创建 Connection 和 Session。
2. 从 InitialContext 获取 `Destination`（本例中为 `Queue`）。
3. 从 connectionFactory 创建 `JMSContext`。
4. 使用 `JMSProducer` 向消息服务器发送消息。
5. 使用 `JMSConsumer` 从 JMS 接收消息。

## JMS API 的线程安全性

使用 JMS API 编写代码时，了解接口的线程安全性很重要。下表展示了各对象的线程安全行为：

**JMS 2.0 对象的线程安全性**

| JMS 2.0 对象                    | 线程安全（允许多线程并发使用） |
|-------------------------------|------------------------------|
| Destination (Queue, Topic)    | 是                           |
| ConnectionFactory             | 是                           |
| JMSContext                    | 否                           |
| JMSProducer                   | 否                           |
| JMSConsumer                   | 否                           |

**JMS 1.1 对象的线程安全性**

| JMS 1.1 对象                    | 线程安全（允许多线程并发使用） |
|-------------------------------|------------------------------|
| Destination (Queue, Topic)    | 是                           |
| ConnectionFactory             | 是                           |
| Connection                    | 是                           |
| Session                       | 否                           |
| MessageProducer               | 否                           |
| MessageConsumer               | 否                           |
