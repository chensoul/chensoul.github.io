Git 有很多命令。大多数工作流只使用其中的一小部分。造成问题的不是命令本身，而是不知道运行一个命令后你的代码位于哪里。

工作目录、暂存区、本地仓库、远程仓库。每个命令在这些之间移动代码。以下是每个命令的作用：

## 保存你的工作

### git add
将文件从工作目录移动到暂存区。

### git commit
将这些暂存文件保存到本地仓库。

### git push
将你的提交上传到远程仓库。

## 获取项目

### git clone
将整个远程仓库拉取到你的机器。

### git checkout
切换到你特定的分支。

## 同步更改

### git fetch
从远程下载更新但不改变你的文件。

### git merge
集成那些更改。

### git pull
一次性完成两者。

## 安全网

### git stash
你的撤销按钮。它临时保存你的未提交更改，这样你可以切换上下文而不会丢失工作。

### git stash apply
把它们带回来。

### git stash pop
把它们带回来并删除 stash。

## 缓存系统如何出错

下图显示了缓存可能出错的 4 种典型情况及其解决方案。

### 1. 惊群问题（Thunder herd problem）
当缓存中大量键同时过期时发生这种情况。然后查询请求直接击中数据库，使数据库过载。

**缓解方法**：
- 避免为键设置相同的过期时间，在配置中添加随机数
- 只允许核心业务数据击中数据库，防止非核心数据在缓存恢复前访问数据库

### 2. 缓存穿透（Cache penetration）
当键在缓存或数据库中都不存在时发生这种情况。应用无法从数据库检索相关数据来更新缓存。这个问题给缓存和数据库都带来很大压力。

**解决方案**：
- 为不存在的键缓存空值，避免击中数据库
- 使用布隆过滤器首先检查键是否存在，如果键不存在，我们可以避免击中数据库

### 3. 缓存击穿（Cache breakdown）
这类似于惊群问题。当热键过期时发生。大量请求击中数据库。由于热键占用 80% 的查询，我们不为它们设置过期时间。

### 4. 缓存崩溃（Cache crash）
当缓存宕机且所有请求都去数据库时发生这种情况。有两种方法可以解决这个问题：
- 设置断路器，当缓存宕机时，应用服务不能访问缓存或数据库
- 为缓存设置集群以提高缓存可用性

## 顶级网络攻击解释

大多数攻击遵循一系列步骤。理解每个步骤使得更容易发现哪里可以检测或预防。

以下是常见攻击如何展开的快速分解：

### 1. 网络钓鱼（Phishing）
攻击者发送假链接指向伪造的登录页面。受害者输入凭据，攻击者捕获它们，并使用它们访问真实系统。

### 2. 勒索软件（Ransomware）
受害者打开恶意附件或文件。勒索软件加密本地数据并要求付款以恢复访问。文件保持锁定直到支付赎金或恢复备份。

### 3. 中间人攻击（Man-in-the-Middle, MitM）
攻击者将自己定位在受害者和服务器之间，拦截两个方向的流量。双方都检测不到拦截。攻击者可以在数据通过时读取或修改数据。

### 4. SQL 注入（SQL Injection）
恶意 SQL 插入到输入字段中，例如 `studentId=117 OR 1=1`。数据库将其执行为有效查询并返回不应该的数据。单个易受攻击的输入字段可以暴露整个表。

### 5. 跨站脚本（Cross-Site Scripting, XSS）
恶意脚本注入到合法页面中。当另一个用户加载该页面时，他们的浏览器执行脚本。会话令牌、cookie 和私有数据可以这样被盗。

### 6. 零日漏洞利用（Zero-Day Exploits）
攻击者发现供应商尚未发现的漏洞。不存在补丁。攻击运行直到供应商识别问题并发送修复，这可能需要数天或数周。

## AI 如何生成图像

有两种主要方式现代模型生成图像：自回归和扩散。

### 自回归模型
逐块生成图像。

**训练期间**：图像分割成令牌，模型学习一个接一个预测它们，就像文本一样。它在图像令牌上最小化下一令牌预测损失。

**推理期间**：模型一次预测一个图像令牌直到形成完整图像。

### 扩散模型
从纯噪声开始并迭代去噪。

**训练期间**：我们向真实图像添加噪声并训练模型预测该噪声。

**推理期间**：模型从随机噪声开始并迭代去噪成清晰图像。

自回归就像一笔一划按顺序画一只狗。扩散就像从粗略草图（粗略形状）开始，然后逐步添加细节并清理图片。

## 译者总结

这篇文章涵盖了多个技术话题：

**Git 基本命令**：
| 类别 | 命令 | 用途 |
|------|------|------|
| 保存工作 | add、commit、push | 保存和上传更改 |
| 获取项目 | clone、checkout | 下载和切换分支 |
| 同步更改 | fetch、merge、pull | 同步远程更改 |
| 安全网 | stash、apply、pop | 临时保存更改 |

**缓存问题**：
| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 惊群 | 大量键同时过期 | 交错过期时间 |
| 缓存穿透 | 键不存在 | 缓存空值、布隆过滤器 |
| 缓存击穿 | 热键过期 | 不设置过期时间 |
| 缓存崩溃 | 缓存宕机 | 断路器、缓存集群 |

**网络攻击**：
- 网络钓鱼、勒索软件
- 中间人、SQL 注入
- XSS、零日漏洞

**AI 图像生成**：
- 自回归：逐块生成
- 扩散：从噪声去噪

**关键洞察**：
- Git 命令在工作流的不同阶段移动代码
- 缓存问题需要针对性解决方案
- 网络攻击遵循可预测模式
- AI 图像生成有两种主要方法

理解这些概念对日常开发和系统安全至关重要。
