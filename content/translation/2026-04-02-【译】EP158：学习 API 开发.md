## 如何学习 API 开发

学习如何开发 API 是现代开发者的重要技能。以下是你需要学习关于 API 开发的思维导图：

### 1. API 基础
什么是 API、API 类型（REST、SOAP、GraphQL、gRPC 等）和 API vs. SDK。

### 2. API 请求/响应
HTTP 方法、响应代码和头部。

### 3. 认证和安全
认证机制（JWT、OAuth 2、API 密钥、基本认证）和安全策略。

### 4. API 设计和开发
RESTful API 原则包括无状态、基于资源的 URL、版本控制和分页。此外，API 文档工具如 OpenAPI、Postman、Swagger。

### 5. API 测试
测试 API 的工具，如 Postman、cURL、SoapUI 等。

### 6. API 部署和集成
在不同语言（如 JS、Python 和 Java）中消费 API。还与第三方 API（如 Google Maps API 和 Stripe API）合作。了解 API 网关，如 AWS、Kong、Apigee。

## 网络协议依赖

理解网络协议依赖对网络安全和网络至关重要。以下是快速理解：

- **IPv4 和 IPv6** 是所有网络的基础。ICMP 和 ICMPv6 处理诊断，而 IPsec 确保安全通信。

- **TCP 和 UDP** 支持各种协议。SCTP 和 DCCP 服务于特定情况。

- **一些基于 TCP 的协议**是 HTTP、SSH、BGP、RDP、IMAP、SMTP、POP 等。

- **基于 UDP 的协议**是 DNS、DHCP、SIP、RTP、NTP 等。

- **SSL/TLS** 加密 HTTPS、IMAPS 和 SMTPS。

- **LDAP 和 LDAPs** 用于 TCP 上的目录服务，并用 SSL/TLS 保护。

- **QUIC** 是基于 UDP 的 TCP+TLS 替代品，用于更快、加密的连接。

- **MCP 或 Model Context Protocol** 是与 LLM 通信的新兴标准。

## 18 个关键设计模式

模式是常见设计问题的可重用解决方案，导致更顺畅、更高效的开发过程。它们作为构建更好软件结构的蓝图。这些是一些最流行的模式：

### 创建型模式
处理对象创建机制，将客户端代码与具体类解耦。

1. **抽象工厂（Abstract Factory）**：家族创建者 - 创建相关项组
2. **构建器（Builder）**：乐高大师 - 逐步构建对象，保持创建和外观
3. **原型（Prototype）**：克隆制造者 - 创建完全准备好的示例副本
4. **单例（Singleton）**：唯一 - 只有一个实例的特殊类

### 结构型模式
帮助将类和对象组合成更大结构。

5. **适配器（Adapter）**：万能插头 - 连接具有不同接口的事物
6. **桥接（Bridge）**：功能连接器 - 链接对象的工作方式和功能
7. **组合（Composite）**：树构建者 - 形成简单和复杂部分的树状结构
8. **装饰器（Decorator）**：定制器 - 在不改变核心的情况下为对象添加功能
9. **外观（Facade）**：一站式商店 - 用单个简化接口代表整个系统
10. **享元（Flyweight）**：空间节省器 - 高效共享小而可重用的物品
11. **代理（Proxy）**：替身演员 - 代表另一个对象，控制访问或操作

### 行为型模式
专注于对象之间的通信和交互。

12. **责任链（Chain of Responsibility）**：请求接力 - 通过对象链传递请求直到处理
13. **命令（Command）**：任务包装器 - 将请求转为对象，准备行动
14. **迭代器（Iterator）**：集合探索者 - 逐个访问集合中的元素
15. **中介者（Mediator）**：通信中心 - 简化不同类之间的交互
16. **备忘录（Memento）**：时间胶囊 - 捕获和恢复对象状态
17. **观察者（Observer）**：新闻广播员 - 通知类关于其他对象的变化
18. **访问者（Visitor）**：技能客人 - 在不改变类的情况下添加新操作
