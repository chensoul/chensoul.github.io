对于本期文章，我很高兴介绍我们的客座作者 Ethan Evans，一位退休的 Amazon VP，他在创建我们今天使用的一些开创性服务方面发挥了关键作用，如 Prime Video、Amazon Video、Amazon Appstore、Prime Gaming（前身为 Twitch Prime）和 Twitch Commerce。

除了他作为工程领导者的经验，Ethan 还是一位职业成长教练，帮助无数人摆脱困境并提升职业生涯。

我们将与 Ethan 讨论求职中最关键的步骤之一：写一份出色的简历。具体来说，他将涵盖几乎所有人在准备简历时犯的三大错误。他还将深入探讨如何纠正它们。

## Ethan 的背景

Ethan 在这里。我在职业生涯中审查了超过 10,000 份简历，进行了超过 2,500 次面试。在 Amazon 和 Twitch，我教授了一门关于如何面试的内部课程，称为"做出伟大雇佣决策"。我是一名"Bar Raiser"，这意味着我是来自雇佣团队外部指定加入面试的人，有权如果我认为候选人不适合公司就否决雇佣决策。

在我职业生涯早期，尽管我有工程背景，我在三家初创公司管理招聘，有招聘人员直接向我汇报。今天，我指导个人完成他们的求职，我测试了什么有效和无效。所有这些都是说，我知道招聘人员和雇佣经理在简历中寻找什么。

我观察到的一件事是我看到的 90% 简历至少犯了三大错误之一。在本文中，我将带你了解每一个以及如何纠正它，然后我们将涵盖你可以做的小事，将你的简历从好带到惊人。最后，我们将总结如何知道何时停止担心你的简历并继续求职的下一阶段。

## 简历的目标

在我们进入常见错误之前，让我们简要回顾简历的目标，以及招聘人员和雇佣经理在阅读简历时寻找什么。

首先，简历的目标不是让你得到工作；它是让你得到电话。如果简历导致有人联系你，那么它就完成了工作。简历错误的一个驱动因素是试图让它做太多。我们想优化我们的简历让我们获得电话，而不是讲述我们的人生故事。

你可能听说过招聘人员只在几秒钟内略读简历。这是真的。招聘人员和经理承受巨大压力，经理经常将雇佣视为他们必须做的额外事情，除了他们的"真正"工作。因此，简历必须优化以快速抓住他们的注意力。即使这样，我们的目标不是让他们读每个字，而是简单地将它放入要联系的小堆人中，然后他们继续。

## 3 大简历错误

我看到的 90% 简历犯了三个错误之一或更多。如果你能解决这些，你的简历可能足够好让你获得电话。

### 错误 1：缺少目标

没有目标，招聘人员或雇佣经理必须猜测你是否想要他们的工作。如果你是一个有一堆简历的招聘人员，其中一些说明候选人想要像你填补的工作，一些没有，你联系谁？

目标清楚地说明你在下一份工作中寻找什么样的角色，一个出色的目标是公式化且短到一眼就能读完。

高级工程师角色的公式示例："寻求高级工程师角色，我可以应用我已证明的交付规模化服务能力为业务提供有价值创新。"

让我们分解这个。

第一部分简单地说明你寻求的角色。如果你想做与你上一份工作不同的事情，这特别重要。没有上下文，大多数人会假设你想继续做你一直在做的事情，可能带有更高级别头衔和更多薪酬。但如果你想转向管理，或从咨询转向传统企业雇佣，那么你必须清楚说明这一点。否则，人们会继续将你放在由你过去定义的盒子中。

第二部分，"...我可以应用我已证明的 X 和 Y 能力..."给你两次机会说明你角色的前两个理由。一开始，你可以突出将你的简历放入"联系"堆的前两个理由。

第三部分，"...为业务创造 XYZ"显示你在那里帮助经理和公司。虽然你在为自己寻找"好工作"，但经理不专注于"给人好工作"。相反，他们想找到能解决他们问题并帮助他们的人。一开始，你想传达你不仅对自己感兴趣，而且致力于为你的新经理提供价值。

我收到对公式第一部分的反馈，因为一些人担心如果他们清楚说明目标角色，他们会错过其他可能适合的不同级别或头衔的其他角色。这是可能的，但有两个原因最好说明清晰目标。

首先，每个角色有许多候选人。就像搜索结果一样，最好是一个工作的第一选择，而不是十个工作的第七选择。如果前三候选人获得电话，你想获得一个电话而不是在所有下面落线。

其次，你可以总是有多个版本的简历。你可以在一个上说明你寻求"工程领导"角色的目标，在另一个上"产品领导"角色 - 如果你两者都合格。然后你可以对不同类型的角色使用它们。

### 错误 2：糟糕的摘要

让我们看几个例子：

**组织混乱 | 管理变更 | 驱动效率**

我是一个有动力、热情、行动导向的领导者，将问题转化为简化流程。我领导了 1000 万美元产品线的扭转，在我管理产品采购和运营履行部门期间，利润率提高 15000 基点（150%）（从 -125% 到 +25%）。

具有涵盖财务、运营、战略、并购、筹款、人力资源、数据分析和数字营销的广泛经验背景。我是一位多重潜力者，喜欢将广泛经验和知识应用于新情况。我推动自己走出舒适区， tackling 没有先前主题专业知识但有快速了解不同职能人员如何运作能力的项目。我在组织混乱中茁壮成长：发现和连接他人经常看不到的部分，并构建新机制来改善我们工作方式。我的敏捷性、处理模糊性能力、强大商业头脑和对挑战的不懈追求，使我成为经历变更或需要转型的成长组织的完美匹配。

这是我通常在简历上看到的典型开头部分。虽然第一段包含一个大成就的优秀数字示例，但第二段是七整行，候选人只是用积极术语谈论自己。它没有数据支持任何东西，只是大块文本，大多数招聘人员会跳过。

**有效且有成就的高管**，在财务、运营、战略、并购、筹款、人力资源、数据分析和数字营销方面有广泛经验。在高速成长环境中建立公司和团队以规模化盈利能力的成功记录。

这第二个例子短得多，更可能被阅读，但进入一长串不同领域列表，让读者想知道候选人实际上想要什么样的工作。

**目标：大规模提供专业发展，指导向执行领导层过渡。**

已证明对游戏、视频、应用和音乐多个 5 亿美元 + 业务的执行领导。全球领导 800+ 团队经验。技术发明人；60+ 已颁发专利。

这最后一个例子来自我自己的执行教练简历。它用一行解释我提供什么，然后给出两行包含我有效做这个能力的关键成就示例。

### 错误 3：列出活动而非影响

这是我看到简历上最常见的单一错误。

人们写下他们做了什么，描述他们的活动，如他们编写代码、管理一些人，或在这个或那个项目上工作。一些人甚至列出他们最无聊的活动，如创建报告或组织文件。

写下你的工作使你成为工人。你将被面试并 leveled 为可以分配工作做的人。

高级绩效者知道他们因影响而非活动被重视。他们指出他们工作的结果，对团队、公司或业务的影响。

考虑以下区别：

- "作为项目 X 的一部分，编写 Java 代码实现几个关键微服务。"
- "作为项目 X 的一部分，为几个关键微服务编写 Java 代码，使其支持企业环境，导致几个关键新客户购买产品。"

哪个工程师听起来更高级和有价值？

通常可以重写你简历上几乎每个子弹，以描述为什么你的工作重要，而不仅是你做了什么。尝试显示你的存在对团队、产品和底线产生积极差异。

好消息是你可以使用高级技巧使这引人入胜：

- **链接到你的结果**：人们夸大和撒谎在简历上，而没有什么比项目链接更能说"我的工作是真实的"
- **人类眼睛自动被粗体吸引**：你可以选择性粗体几个关键词或短语，确保你最强的成就被注意到
- **使用行动动词**：如'owned'、'built'、'drove'或'delivered'给你的子弹"主动语态"。这表明你主动做某事，而不仅是在发生时被动在场
- **人们相信具体数字**：子弹中有数字结果的简历获得比没有的多 132% 电话。我刚编造了那个观点，但我打赌你相信它
- **可能时包括美元数字**：如果与结果相关

让我们将这个应用到我们的例子，看看它变得多强：

- "为项目 X 用 Java 交付几个关键微服务，使其支持企业环境。这导致服务使用增加 74%，包括几个关键新客户采用，产生 2900 万美元增量收入。"

现在，你认为公司想雇佣谁？

## 译者总结

这篇文章由前 Amazon VP Ethan Evans 分享简历撰写的核心洞察：

**简历目标**：
- 不是得到工作，而是得到电话
- 优化让招聘人员将你放入"联系"堆
- 招聘人员几秒内略读简历

**3 大简历错误**：

1. **缺少目标**
   - 清楚说明寻求的角色
   - 公式：角色 + 能力 + 业务价值
   - 可以有多个版本针对不同角色

2. **糟糕的摘要**
   - 避免大块文本
   - 用数据支持成就
   - 短小精悍，聚焦关键成就

3. **列出活动而非影响**
   - 展示影响而非活动
   - 使用行动动词（owned、built、drove）
   - 包含具体数字和美元金额
   - 链接到实际项目

**好简历特征**：
- 清晰目标（一眼读懂）
- 数据支持的成就
- 行动动词
- 具体数字（132% 更多电话）
- 项目链接

**关键洞察**：
- 高级绩效者因影响被重视
- 数字结果使简历可信
- 针对性优于通用性
- 从用户（招聘人员）视角设计

理解这些原则对创建有效简历至关重要。
