本周的系统设计复习：

**12 个 Claude Code 功能**

1. **CLAUDE.md**：项目记忆文件，用于定义自定义规则和约定。Claude 在每次会话开始时读取。
2. **Permissions（权限）**：控制 Claude 可以/不可以使用的工具。
3. **Plan Mode（计划模式）**：Claude 先计划再行动。你可以在任何代码更改之前审查计划。
4. **Checkpoints（检查点）**：项目的自动快照，如果出现问题可以回滚。
5. **Skills（技能）**：可复用的指令文件，Claude 自动遵循。
6. **Hooks（钩子）**：在生命周期事件（如 PreToolUse 或 PostToolUse）上运行自定义 Shell 脚本。
7. **MCP**：将 Claude 连接到任何外部工具，如数据库和第三方服务。
8. **Plugins（插件）**：使用第三方集成扩展 Claude，包含技能、MCP 和钩子。
9. **Context（上下文）**：为 Claude 提供所需内容，并使用 `/context` 管理当前上下文窗口。
10. **Slash Commands（斜杠命令）**：为你经常运行的任务创建快捷方式。输入 `/` 并从保存的命令中选择。
11. **Compaction（压缩）**：压缩长对话以节省 tokens。
12. **Subagents（子代理）**：为复杂任务生成并行代理。拆分大型多步骤工作流并同时运行它们。

## 译者总结

### 一句话核心
ByteByteGo 第 209 期介绍了 Claude Code 的 12 个核心功能，涵盖权限控制、计划模式、钩子、MCP 集成等实用特性。

### 要点回顾
- **CLAUDE.md**：项目级自定义规则文件，每次会话自动读取
- **Plan Mode**：先审查计划再执行代码更改，适合谨慎场景
- **Hooks**：在工具使用前/后等生命周期事件触发自定义脚本
- **MCP/Plugins**：扩展 Claude 与外部工具、第三方服务集成
- **Subagents**：并行执行复杂多步骤任务

### 边界与易误读
- 这是 ByteByteGo 新闻通讯的摘要内容，非深度教程
- 部分功能（如 MCP、Plugins）需要额外配置

### 术语与延伸
- **MCP**：Model Context Protocol，用于连接外部工具的标准协议
- **Hooks**：事件驱动的脚本执行机制，类似 Git Hooks
- **Compaction**：对话压缩技术，解决长上下文 token 消耗问题
