## 现代软件栈的 9 层

在当今世界，构建软件意味着跨多层工作，每层有自己的角色、工具和技术。以下是构成大多数现代应用的 9 层：

### 1. 表示层（UI/UX）
处理用户如何与应用交互，专注于视觉、布局和可用性。

### 2. 边缘和交付（可选）
通过全球交付网络将内容更靠近用户，减少延迟并提高性能。

### 3. 集成层（API）
定义系统不同部分如何通信，实现组件间互操作性。

### 4. 消息和异步处理（可选）
在后台处理任务和事件以提高可扩展性和响应能力。

### 5. 业务逻辑层
实现应用的核心规则、工作流和决策过程。

### 6. 数据访问层
充当应用逻辑和存储数据之间的桥梁，确保安全高效检索或更新。

### 7. 数据存储层
存储、组织和管理应用的结构化和非结构化数据。

### 8. 分析和 ML（可选）
分析数据以生成洞察、预测和智能功能。

### 9. 基础设施层（托管/运行时）
提供计算环境和资源用于部署、运行和扩展应用。

## 并发不是并行

### 并发
这是一种设计方法，任务可以启动、运行和在重叠期间完成，即使在单个 CPU 核心上。它是关于同时管理多个任务。

CPU 在任务间快速切换（上下文切换），创建任务同时进展的错觉，尽管它们不是。

并发非常适合涉及等待的任务，如 I/O 操作。它允许其他任务在等待期间进展，提高整体效率。

### 并行
指使用多个 CPU 核心同时执行多个任务。

并行擅长重计算，如数据分析或渲染图形，其中任务可以划分并在不同核心上同时运行。

### 它们如何一起工作

注意并发和并行是不同概念，但它们密切相关。设计良好的并发程序可以扩展以在需要时使用多核进行并行。

通过理解并发和并行之间的差异和相互作用，我们可以设计更高效系统并创建更好性能的应用。

## JWT vs. PASETO

基于令牌的认证多年来变得相当流行。传统上，JWT 主导这个空间。

但现在，一个新玩家正在掀起波澜：PASETO，或平台无关安全令牌。

那么，两者之间有什么区别？

### JWT
JWT 或 JSON Web 令牌是一个开放标准，用于在双方之间安全传输信息。JWT 由头部、负载和签名组成。

JWT 可用于实现客户端和服务器应用之间的无状态认证。

### PASETO
PASETO 是 JWT 的现代替代方案。它通过实现安全默认值解决 JWT 的安全缺陷。与 JWT 不同，PASETO 强制强大、密码学健全的算法，减少漏洞风险。

PASETO 通常由版本、目的和负载组成。有两种类型的 PASETO：

- **公共 PASETO**：它们使用非对称加密签名，确保数据完整性，但不确保其机密性
- **本地 PASETO**：它们使用对称加密算法加密，确保令牌内包含数据的机密性

## Linux Cron 速查表

Cron 是一个基于时间的作业调度器，允许用户通过在指定间隔运行命令或脚本来自动化重复任务。它使用 Cron 表达式，一个定义调度的语法，由五个字段组成（分钟、小时、日、月、工作日）。

Cron 广泛用于系统维护、备份、日志轮转和自动化任务，如在定期发送电子邮件或运行脚本。在此速查表中，我们涵盖：

## AI 代理 vs. MCP

### AI 代理
AI 代理是一个软件程序，可以与环境交互、收集数据，并使用这些数据实现预定目标。AI 代理可以选择执行最佳行动以实现这些目标。

**AI 代理的关键特征**：

- **代理可以执行自主行动**
  无需持续人类干预。此外，它们可以有人类在循环中保持控制。

- **代理有记忆**
  存储个人偏好并允许个性化。它也可以存储知识。LLM 可以进行信息处理和决策功能。

- **代理必须能够感知和处理环境中可用的信息**

### Model Context Protocol（MCP）
MCP 是 Anthropic 引入的一个新系统，使 AI 模型更强大。

它是一个开放标准，允许 AI 模型（如 Claude）连接到数据库、API、文件系统和其他工具，而无需为每个新集成编写自定义代码。

**MCP 遵循客户端 - 服务器模型，有 3 个关键组件**：

1. **Host（主机）**
   AI 应用如 Claude

2. **MCP Client（MCP 客户端）**
   AI 模型（如 Claude）内部的组件，允许它与 MCP 服务器通信

3. **MCP Server（MCP 服务器）**
   连接 AI 模型到外部系统的中间人

## 译者总结

这篇文章涵盖了软件栈、并发、认证、Cron 和 AI 代理的核心概念：

**现代软件栈 9 层**：
| 层 | 职责 | 可选 |
|------|------|------|
| 表示层 | UI/UX | 否 |
| 边缘交付 | CDN、降低延迟 | 是 |
| 集成层 | API 通信 | 否 |
| 消息异步 | 后台任务 | 是 |
| 业务逻辑 | 核心规则 | 否 |
| 数据访问 | 数据检索 | 否 |
| 数据存储 | 存储管理 | 否 |
| 分析 ML | 洞察预测 | 是 |
| 基础设施 | 托管运行时 | 否 |

**并发 vs. 并行**：
| 特性 | 并发 | 并行 |
|------|------|------|
| 定义 | 管理多任务 | 同时执行 |
| CPU 核心 | 单核也可 | 需要多核 |
| 适用 | I/O 等待任务 | 重计算任务 |
| 机制 | 上下文切换 | 同时运行 |

**JWT vs. PASETO**：
| 特性 | JWT | PASETO |
|------|-----|--------|
| 组成 | 头部、负载、签名 | 版本、目的、负载 |
| 安全 | 可配置 | 强制安全默认 |
| 类型 | - | 公共、本地 |
| 加密 | 可选 | 强制 |

**AI 代理 vs. MCP**：
| 特性 | AI 代理 | MCP |
|------|--------|-----|
| 定义 | 自主软件程序 | 开放标准 |
| 功能 | 感知、决策、行动 | 连接 AI 到外部工具 |
| 组件 | 记忆、感知 | Host、Client、Server |

**关键洞察**：
- 现代软件需要多层架构
- 并发和并行有不同用途
- PASETO 提供更强安全默认
- Cron 自动化重复任务
- MCP 简化 AI 与工具集成

理解这些概念对构建现代软件系统至关重要。
